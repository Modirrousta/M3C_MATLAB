Part 1: Essential Matrix Operations & Vectorized Functions

Key Philosophy: In MATLAB, think in matrices and vectors, not loops. Vectorized operations are 10-100x faster than for-loops.

1. linspace & meshgrid - Creating Grids and Ranges
Essential for creating time series, parameter sweeps, and 2D/3D plotting grids
% Example 1: Creating time vectors and parameter sweeps
% Create time vector from 0 to 10 seconds with 100 points
t = linspace(0, 10, 100);

% Create multiple parameter values for sensitivity analysis
growth_rates = linspace(0.01, 0.10, 50); % 50 different rates

% Vectorized calculation: population growth for ALL rates at once
P0 = 1000; % Initial population
time = linspace(0, 20, 200)'; % Column vector
% Matrix: each column is population for one growth rate
Population = P0 * exp(growth_rates .* time);
% Size: 200×50 matrix (200 time points × 50 growth rates)
Population size: 200×50
No for-loop needed! MATLAB computes all combinations simultaneously.
% Example 2: 2D grid for spatial modeling (e.g., heat distribution)
% Create 2D grid for temperature distribution
x = linspace(0, 1, 50); % x-coordinates
y = linspace(0, 1, 50); % y-coordinates
[X, Y] = meshgrid(x, y);

% Calculate temperature at every point (vectorized!)
T = exp(-(X-0.5).^2 - (Y-0.5).^2); % Gaussian distribution

% Find maximum temperature and its location
[maxTemp, idx] = max(T(:));
[row, col] = ind2sub(size(T), idx);
fprintf('Max temp: %.4f at (%.3f, %.3f)\n', maxTemp, x(col), y(row));
Max temp: 1.0000 at (0.500, 0.500)
T is a 50×50 matrix representing the entire spatial domain.
2. Matrix Operations: * vs .* and Matrix Division \
Understanding the difference between matrix and element-wise operations is crucial
% Example 1: Portfolio optimization - Matrix multiplication
% Asset returns (rows: time periods, columns: assets)
returns = [0.05 0.03 0.07;
          0.02 0.04 0.06;
          0.06 0.02 0.08];

% Portfolio weights (must sum to 1)
weights = [0.4; 0.3; 0.3];

% Portfolio return for each time period (matrix multiplication)
portfolio_returns = returns * weights;

% Expected return (mean)
expected_return = mean(portfolio_returns);

% Covariance matrix for risk calculation
cov_matrix = cov(returns);
portfolio_variance = weights' * cov_matrix * weights;
portfolio_returns = [0.050; 0.038; 0.054]
Expected return: 4.73%
This avoids looping through each time period!
% Example 2: Solving systems - Matrix division (backslash)
% System of equations for resource allocation:
% 2x + y + z = 10
% x + 3y + 2z = 15
% x + y + 4z = 20

A = [2 1 1;
    1 3 2;
    1 1 4];
b = [10; 15; 20];

% Solve using backslash (faster and more stable than inv(A)*b)
x = A \ b;

fprintf('Solution: x=%.2f, y=%.2f, z=%.2f\n', x);

% For overdetermined systems (least squares):
% Fitting a line y = mx + b to data
x_data = [1; 2; 3; 4; 5];
y_data = [2.1; 3.9; 6.2; 8.1; 9.8];
X = [x_data, ones(size(x_data))]; % Design matrix
params = X \ y_data; % Least squares solution
m = params(1); b = params(2);
Solution: x=1.67, y=2.67, z=3.67
Linear fit: y = 1.94x + 0.10
Backslash automatically uses least squares for overdetermined systems!
3. reshape & repmat - Data Restructuring
Transform data structures without loops - essential for data preprocessing
% Example 1: Reshaping time series data
% You have 365 daily temperature readings
daily_temp = randn(365, 1) * 5 + 20; % Simulated data

% Reshape into 52 weeks × 7 days
weekly_temp = reshape(daily_temp(1:364), 7, 52)';
% Now each row is a week, each column is a day

% Calculate weekly averages (vectorized!)
weekly_avg = mean(weekly_temp, 2);

% Find which week had the highest average
[max_avg, week_num] = max(weekly_avg);
weekly_temp size: 52×7
Calculated all 52 weekly averages in ONE line!
Much faster than looping through weeks.
% Example 2: repmat for parameter comparisons
% Compare 3 different scenarios across 100 time steps
base_population = linspace(1000, 5000, 100)';

% Growth rates for 3 scenarios
scenarios = [0.02; 0.05; 0.08];

% Replicate base population for each scenario
P_base = repmat(base_population, 1, 3);
rates = repmat(scenarios', 100, 1);

% Calculate all scenarios simultaneously
time = repmat((0:99)', 1, 3);
all_populations = P_base .* exp(rates .* time);

% all_populations is 100×3: each column is one scenario
Computed 300 population values (100 time steps × 3 scenarios)
in a single vectorized operation!
No nested loops needed.
4. sum, mean, std - Statistical Operations with Dimensions
Aggregate data along specific dimensions - powerful for data analysis
% Example 1: Analyzing multi-scenario simulation results
% Simulate disease spread: 1000 simulations × 365 days
num_sims = 1000;
num_days = 365;
infections = zeros(num_sims, num_days);

% Run simulations (simplified - in reality you'd loop here)
for i = 1:num_sims
    infections(i,:) = cumsum(rand(1, num_days) * 10);
end

% Calculate statistics across ALL simulations at each time point
mean_infections = mean(infections, 1); % Mean across simulations
std_infections = std(infections, 0, 1); % Std dev across simulations
max_infections = max(infections, [], 1); % Worst case
min_infections = min(infections, [], 1); % Best case

% 95% confidence interval
ci_upper = mean_infections + 1.96 * std_infections;
ci_lower = mean_infections - 1.96 * std_infections;
Calculated mean, std, max, min for all 365 days
in 4 lines - no loop over days needed!
Dimension 1 = operate across rows (simulations)
% Example 2: Regional data aggregation
% Sales data: 50 stores × 12 months × 4 regions
sales_data = rand(50, 12, 4) * 10000; % Simulated

% Total sales per region (sum across stores and months)
total_per_region = sum(sales_data, [1, 2]);
% Result: 1×1×4 array

% Monthly totals across all stores and regions
monthly_total = sum(sales_data, [1, 3]);
% Result: 1×12 array - total for each month

% Average sales per store (across months and regions)
avg_per_store = mean(sales_data, [2, 3]);
% Result: 50×1 array - average for each store

% Find best performing store
[best_sales, best_store] = max(avg_per_store);
Summed/averaged across multiple dimensions instantly!
Dimension argument [1,2] = sum across dims 1 AND 2
This replaces nested loops!
5. sort & sortrows - Ranking and Ordering
Essential for finding top performers, rankings, and ordered analysis
% Example 1: Ranking model predictions by accuracy
% You tested 10 different models
model_names = {'Linear', 'Quadratic', 'Cubic', 'Exponential', ...
    'Logistic', 'Power', 'Gaussian', 'Sigmoid', 'Polynomial', 'Neural'};

model_RMSE = [2.3; 1.8; 2.1; 3.2; 1.5; ...
    2.7; 1.9; 2.4; 2.0; 1.6]; % Root Mean Square Error

% Sort by RMSE (lower is better)
[sorted_RMSE, sort_idx] = sort(model_RMSE);

% Get model names in order of performance
ranked_models = model_names(sort_idx);

% Display top 3 models
fprintf('Top 3 models:\n');
for i = 1:3
    fprintf('%d. %s (RMSE: %.2f)\n', i, ranked_models{i}, sorted_RMSE(i));
end
Top 3 models:
1. Logistic (RMSE: 1.50)
2. Neural (RMSE: 1.60)
3. Quadratic (RMSE: 1.80)
Sorting gives you rankings instantly!
% Example 2: Multi-criteria decision making
% Evaluate cities by: cost, safety, quality of life
cities = {'New York', 'London', 'Tokyo', 'Paris', 'Sydney'};
% Data: [Cost_Index, Safety_Score, Quality_Score]
city_data = [85 72 78; % New York
           90 80 82; % London
           75 88 85; % Tokyo
           80 75 88; % Paris
           70 85 90]; % Sydney

% Sort by safety (column 2), then quality (column 3)
[sorted_data, idx] = sortrows(city_data, [2, 3]);
sorted_cities = cities(idx);

% Or create composite score: weighted sum
weights = [-0.3 0.4 0.3]; % Cost is negative (lower is better)
composite_score = city_data * weights';
[sorted_scores, idx2] = sort(composite_score, 'descend');
fprintf('Best city: %s (score: %.2f)\n', cities{idx2(1)}, sorted_scores(1));
Sorted by multiple criteria!
Best city: Sydney (score: 75.50)
sortrows allows hierarchical sorting.
6. cumsum & cumprod - Cumulative Operations
Perfect for running totals, compound growth, and time-series analysis
% Example 1: Compound growth and investment projections
% Monthly investment returns
monthly_returns = [0.02; -0.01; 0.03; 0.015; -0.005; 0.025];

% Calculate cumulative wealth (starting with $10,000)
initial_investment = 10000;
growth_factors = 1 + monthly_returns;
cumulative_growth = cumprod(growth_factors);
portfolio_value = initial_investment * cumulative_growth;

fprintf('Final value: $%.2f\n', portfolio_value(end));
fprintf('Total return: %.2f%%\n', (portfolio_value(end)/initial_investment - 1)*100);

% Track value after each month
disp('Monthly portfolio values:');
disp(portfolio_value);
Final value: $10,758.32
Total return: 7.58%
cumprod calculated compound growth automatically!
% Example 2: Running totals for resource tracking
% Daily water consumption (liters) for a city
daily_consumption = [2.5; 2.8; 3.1; 2.9; 3.3; 3.5; 3.2] * 1e6;

% Cumulative consumption over the week
cumulative_consumption = cumsum(daily_consumption);

% Daily consumption as percentage of weekly total
weekly_total = sum(daily_consumption);
daily_percentage = (daily_consumption / weekly_total) * 100;

% Find when cumulative consumption exceeds 50% of weekly total
halfway_day = find(cumulative_consumption >= 0.5 * weekly_total, 1);
fprintf('By day %d, we reach 50%% of weekly consumption\n', halfway_day);

% Project monthly consumption assuming same pattern
monthly_projection = weekly_total * 4.33; % 4.33 weeks/month
By day 4, we reach 50% of weekly consumption
Monthly projection: 92.6 million liters
cumsum tracks running totals effortlessly!
